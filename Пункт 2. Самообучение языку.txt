**Пункт 2. Самообучение языку**

На этом этапе ЕВА, обладая единственным слоем `PrimordialLayer` со случайными весами, должна освоить базовые закономерности языка — научиться предсказывать следующее слово в тексте. Процесс полностью автоматизирован и не требует внешней разметки. Система самостоятельно оценивает свой прогресс и принимает решение о завершении этапа.

### 2.1. Цели этапа

- Сформировать в матрицах слоя способность к языковому моделированию (Causal LM).
- Накопить первые осмысленные слепки состояний в хранилище.
- Проверить работоспособность SRG и механизма любопытства в учебном цикле.
- Достичь средней уверенности выше порогового значения на контрольной выборке.

### 2.2. Данные для обучения

На этом этапе используется только сырой текст — русская Википедия. Никакой разметки, никаких инструкций. Система учится исключительно на задаче предсказания следующего токена.

Загрузка данных выполняется через `DataManager.load_wikipedia()`. Тексты разбиваются на блоки фиксированной длины (512 токенов). Для токенизации используется стандартный BPE-токенизатор, обученный на русском корпусе, либо собранный из `PrimordialLayer`.

### 2.3. Цикл обучения

Обучение организовано как бесконечный цикл, который прерывается только при достижении критерия остановки. На каждой итерации:

**2.3.1. Извлечение батча.** Из Википедии извлекается случайный блок текста объёмом 512 токенов. Блок преобразуется в тензоры `input_ids` (первые 511 токенов) и `labels` (сдвиг на 1 позицию).

**2.3.2. Прямой проход.** Батч пропускается через `PrimordialLayer`. Вычисляется кросс-энтропийная потеря между логитами и целевыми токенами.

**2.3.3. Обратный проход.** Градиенты распространяются через слой, обновляя матрицы `W_Q`, `W_K`, `W_V`, `W_O`, `W_up`, `W_gate`, `W_down`. Используется оптимизатор AdamW с learning rate 1e-4.

**2.3.4. Оценка SRG.** На каждом N-м шаге (например, каждые 100 итераций) выполняется пробная генерация: модель получает начальный фрагмент текста и генерирует продолжение. Для этого используется авторегрессионный цикл: на каждом шаге модель предсказывает следующий токен, добавляет его к входной последовательности и повторяет процесс, пока не будет достигнута максимальная длина или сгенерирован EOS-токен. Контекстный вектор `c_query` вычисляется из начального фрагмента; `c_response` — из скрытого состояния сгенерированного продолжения. SRG вычисляет уверенность по стандартной формуле.

**2.3.5. Сохранение слепка.** Если уверенность превышает порог (0.8), текущее состояние (Key и Value для последнего токена) сохраняется в `StateStorage`. Одновременно контекстный вектор добавляется в FAISS-индекс.

**2.3.6. Анализ любопытства.** Если уверенность ниже 0.6, увеличивается счётчик `curiosity_counter`. При достижении порога `CURIOSITY_THRESHOLD` (10) система генерирует уточняющий вопрос, используя механизм `CuriosityLoop.generate_clarification()`. Вопрос записывается в лог, но на этом этапе не отправляется пользователю (обучение идёт автономно). Счётчик сбрасывается.

**2.3.7. Сохранение чекпоинта.** Каждые N итераций (например, 1000) веса слоя, FAISS-индекс и метаданные сохраняются на диск.

### 2.4. Критерий остановки

Обучение продолжается до тех пор, пока одновременно не выполнятся три условия:

1. Средняя уверенность SRG за последние 500 оценок превышает 0.7.
2. Количество успешных слепков в хранилище превышает 1000.
3. Счётчик любопытства не срабатывал в течение последних 500 итераций (модель перестала систематически ошибаться).

При достижении критерия обучение останавливается, и система переходит к этапу инструктивного дообучения.

### 2.5. Автоматизация процесса

Весь описанный цикл реализуется в методе `train_language(self, data_iterator, max_steps=None)`, который принимает итератор данных и опциональное максимальное число шагов. Метод возвращает статистику обучения: итоговую среднюю уверенность, количество сохранённых слепков, финальное значение потерь.

### 2.6. Ожидаемый результат

По завершении этапа ЕВА способна:
- порождать грамматически связный текст (хотя и не обязательно осмысленный);
- сохранять успешные состояния для часто встречающихся контекстов;
- оценивать собственные ответы через SRG;
- фиксировать неудачные попытки через счётчик любопытства.

Накопленные слепки станут основой для более быстрой обработки похожих запросов на следующих этапах, а механизм любопытства подготовит систему к активному диалоговому обучению.
