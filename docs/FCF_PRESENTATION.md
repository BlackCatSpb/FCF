# FCF — Что это и зачем

## Простыми словами

Представь ребёнка. Он не рождается со знанием энциклопедии. У него есть мозг (архитектура), способность учиться (алгоритмы) и врождённое чувство «правильно/неправильно» (этика). Всё остальное он узнаёт сам, взаимодействуя с миром.

**FCF — это такой ребёнок, только в компьютере.**

---

## В чём отличие от ChatGPT и других ИИ

| Обычный ИИ | FCF |
|------------|-----|
| Ему дали готовую модель, обученную на всём интернете | Начинает с чистого листа — случайные числа |
| Размер модели фиксирован навсегда | Растёт когда нужно, сжимается когда не нужно |
| Знания хранятся в миллиардах чисел, непонятно где именно | Знания хранятся как ПРАВИЛА — «если вижу А в контексте Б, делаю В» |
| Не может объяснить почему ответил так | Каждое знание имеет описание и историю происхождения |
| Этику можно «отменить» хитрым промптом | Этика встроена в архитектуру — нельзя удалить |
| Обучили → используют. Всё. | Продолжает учиться во время работы |

## Как это работает — на примере

### «Золотая рожь» и «золотая статуэтка»

Слово «золотая» — одно и то же. Но смысл разный. Почему?

В FCF каждое понятие — не просто число, а **правило**: как оно взаимодействует с соседями в разных контекстах. «Золотая + рожь» активирует правило «цвет злака». «Золотая + статуэтка» — правило «материал предмета». Система не запоминает все комбинации — она выводит правила один раз, а потом применяет их к любым новым словам.

Это и есть **StateGrammar** — 41 правило того, как смыслы сочетаются друг с другом.

### Обучение как у ребёнка

1. Система видит текст: «История — это наука о прошлом»
2. Пытается предсказать следующее слово
3. Ошибается → исправляет свои матрицы
4. Каждые 1000 шагов — проверяет качество генерации
5. Если качество растёт — сохраняет чекпоинт
6. Если падает — запускает само-коррекцию (KCA)
7. Параллельно выводит ПРАВИЛА: «"наука" часто идёт после "это" в определениях»
8. Ночью (Sleep Mode) — пересматривает старые знания, удаляет устаревшее, открывает новые концепты

### Что происходит когда вы задаёте вопрос

```
Вы: "Что такое квантовая запутанность?"

FCF:
1. Понимает что это вопрос о физике (контекстный вектор)
2. Ищет в памяти похожие запросы (FAISS + HNSW)
3. Если знает ответ — генерирует сразу (SRG > 0.8)
4. Если не уверен — запускает KCA: 5 попыток улучшить ответ
5. Если всё равно не знает — честно говорит: "Я не уверен"
6. Сохраняет ваш вопрос в "список пробелов в знаниях"
7. Ночью идёт в Wikipedia, читает статью, учит новый домен
8. В следующий раз — уже знает ответ
```

## Что FCF умеет делать (41 механизм StateGrammar)

Измерять **интенсивность** смысла: «очень золотая» ≠ «слегка золотистая»  
Предсказывать **следующее состояние** в цепочке: «я вижу... → значит...»  
Понимать **отрицание**: «не золотая» — это не отсутствие, а свой смысл  
Держать **суперпозицию смыслов**: пока нет контекста, «золотая» = и цвет, и материал  
Проверять **осмысленность комбинаций**: «квадратный круг» → невалидно  
Строить **иерархию**: «рожь» → «злак» → «растение» → «живое»  
**Открывать новые концепты**: если «A+B» встретилось 10 раз → новый концепт  
Измерять **расстояние между смыслами** не по векторам, а по правилам трансформации  
Находить **что сохраняется при композиции**: «золотая рожь» — всё ещё рожь  
Обнаруживать **самореферентность**: «это предложение ложно»  
Измерять **информационный прирост**: насколько неожиданна композиция  

Понимать **причинность**: «если бы не А, то не Б»  
Различать **время**: прошлое, настоящее, будущее, гипотетическое  
Моделировать **знание о знании**: «я знаю что я не знаю X»  
Применять **кванторы**: «все X», «некоторые X», «ни один X»  
Находить **резонанс**: два смысла усиливают друг друга  
Работать с **границами**: «почти А, наполовину Б»  
Спускаться по **градиенту потенциала**: поиск лучшего состояния  
Измерять **устойчивость**: какие свойства выживают при помехах  
Применять **теорию категорий**: функторы между пространствами смыслов  
Вычислять **геодезические**: кратчайший путь между смыслами  

**Модифицировать свои правила**: стало хуже → меняю правило  
Разрешать **противоречия**: thesis + antithesis → synthesis  
**Догадываться о причинах**: вижу эффект → какая причина?  
Находить **аналогии**: «A относится к B как C относится к D»  
**Сочинять невиданное**: zero-shot композиция из правил  
Проверять **фрактальную самоподобность**: правило работает на всех уровнях  
Понимать **цели**: не «как», а «зачем»  
Анализировать **сюжеты**: tension → climax → resolution  
Измерять **эмоции**: радость, грусть, гнев, страх, удивление...  
Строить **альтернативные миры**: «что если X, тогда Y, тогда Z...»  

Учитывать **культурный контекст**: «свобода» в русском ≠ freedom  
**Мечтать**: сюрреалистическая рекомбинация без ограничений  
Решать **этические дилеммы**: утилитаризм vs деонтология  
Вести **экономику состояний**: стоимость создания, ценность, амортизация  
**Эволюционировать**: генетический алгоритм — мутации, скрещивание, отбор  
Искать **равновесие Нэша**: конкуренция смыслов за контекст  
Распределять **внимание**: ограниченный бюджет → что важнее?  
Создавать **метафоры**: «спор — это война» (Lakoff)  
Осознавать **глубину самосознания**: «я знаю что я знаю что X»  
Искать **новое из любопытства**: энтропия как драйвер

## Что нужно чтобы запустить

```bash
# Установка
pip install -r requirements.txt

# Запуск (двойной клик по FCF.lnk тоже работает)
python run.py --lazy-learn --checkpoint checkpoints\language\step_023000
```

После запуска система:
- Загружает модель (233 млн параметров)
- Начинает учиться на Wikipedia в фоне
- Каждые 1000 шагов выводит правила композиции
- Отвечает на вопросы в консоли
- Ночью проводит консолидацию знаний

**Команды в консоли:** `grammar` (посмотреть правила), `stats` (статистика), `discover` (найти новые правила), `bench` (история прогресса), `train` (форсировать обучение).
