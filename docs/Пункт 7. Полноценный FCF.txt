**Пункт 7. Полноценный FCF — Интеграция State Algebra, KCA‑цикла и иерархического HNSW**

На этом этапе ЕВА, прошедшая путь от случайных весов до многослойной системы с доменными адаптерами и механизмами консолидации, обретает свою окончательную форму — **Fractal Cognitive Fabric**. Три ключевых компонента интегрируются в архитектуру, превращая её из самообучающейся системы в полноценную когнитивную среду, способную к композиции знаний, гарантированной сходимости рассуждений и эффективному поиску состояний.

### 7.1. Цели этапа

- Реализовать State Algebra — набор дифференцируемых операторов, позволяющих комбинировать латентные коды различных состояний.
- Внедрить KCA‑цикл — механизм итеративного уточнения состояния с утилитарной функцией и протоколом гарантированной сходимости.
- Построить иерархический HNSW‑индекс для эффективного поиска состояний на разных уровнях абстракции.
- Сформировать полный трёхфазный когнитивный цикл, объединяющий восприятие, порождение и исполнение.

### 7.2. State Algebra — композиция знаний без экспоненциального роста

State Algebra решает фундаментальную задачу: как из двух существующих состояний (например, «квантовая механика» и «музыка») породить третье («объясни квантовую запутанность через аналогию с музыкой») без сохранения каждого возможного сочетания в памяти.

**7.2.1. Пространство латентных кодов.** Каждое матричное состояние представлено латентным кодом `z ∈ ℝᵏ`. Успешные коды образуют многообразие `ℳ_z` — подмножество ℝᵏ, соответствующее валидным, проверенным состояниям.

**7.2.2. Операторы композиции.** Реализуются четыре оператора, работающие непосредственно в пространстве латентов:

- **Суммирование:** `z_{A+B} = Proj_ℳ(z_A + z_B)` — для аддитивных контекстов («расскажи о физике и лирике»).
- **Масштабирование:** `z_{αA} = Proj_ℳ(α · z_A)` — для усиления или ослабления аспекта.
- **Вычитание:** `z_{A\B} = Proj_ℳ(Clamp(z_A − β · z_B, −τ, τ))` — для фильтрации нежелательных аспектов.
- **Кросс‑аттеншн композиция:** `z_{comp} = Proj_ℳ(TransformerBlock([z_A; z_B]))` — для сложных нелинейных взаимодействий.

Здесь `Proj_ℳ` — проектор на многообразие валидных состояний, реализованный как лёгкий автоэнкодер, обученный на истории успешных латентных кодов.

**7.2.3. Обучение операторов.** Операторы обучаются на синтетических парах `(z_A, z_B) → z_{target}`, где `z_{target}` — оптимальный латентный код для комбинированного контекста. Целевой код находится тем же KCA‑циклом (см. 7.3). Обучение не требует внешнего учителя: достаточно двух существующих состояний и запроса на их комбинацию.

**7.2.4. Использование в цикле генерации.** При поступлении сложного запроса система:
1. Разбивает запрос на семантические компоненты (через спектральную кластеризацию токенов).
2. Для каждого компонента извлекает соответствующий латентный код из `DomainRegistry` или `StateStorage`.
3. Применяет подходящий оператор State Algebra для композиции результирующего кода.
4. Разворачивает композитный код в матрицы `ΔW` и выполняет генерацию.

Это позволяет системе порождать новые знания «на лету», не сохраняя каждую комбинацию в памяти.

### 7.3. KCA‑цикл — итеративное уточнение с гарантированной сходимостью

Knowledge‑Conscious Attention (KCA) — это механизм, который заменяет простую рефлексию (Inner Critic) формальным итеративным процессом с математическими гарантиями.

**7.3.1. Утилитарная функция.** Цель KCA — не реконструкция входа, а максимизация качества ответа. Целевая функция:

```
ℒ_KCA(z) = −λ₁ · SRG_conf(c_out, p) + λ₂ · D_KL(p ∥ p_target) + λ₃ · ∥c_out − c_target∥²
```

где:
- `SRG_conf` — уверенность Semantic Relevance Gate;
- `p` — распределение логитов модели;
- `p_target` — эталонное распределение домена (центроид кластера или вывод critic‑сети);
- `c_out` — контекстный вектор сгенерированного ответа;
- `c_target` — целевой контекстный вектор (например, центроид соответствующего домена).

**7.3.2. Градиентный спуск по латентному коду.** Латентный код оптимизируется непосредственно:

```
z_{t+1} = z_t − η_t · ∇_z ℒ_KCA(z_t)
```

где `η_t = η₀ · ρᵗ` — адаптивное демпфирование (ρ ≈ 0.85), обеспечивающее экспоненциальное затухание шага.

Градиенты проходят через декодеры `ΔW(z)` и через прямой проход LLM, но веса базовой модели остаются замороженными — обновляется только `z`.

**7.3.3. Протокол динамической сходимости.** Гарантирует завершение цикла не более чем за 5 итераций:

1. **Адаптивное демпфирование:** `η_t = η₀ · ρᵗ` — шаг затухает экспоненциально.
2. **Детектор осцилляции:** вычисляется косинусное сходство между последовательными градиентами. Если `cos(∇_t, ∇_{t-1}) < −0.5`, система зафиксировала колебательный режим; тогда последние три значения `z` усредняются.
3. **Монитор гейта:** отслеживается гейт `γ` в формуле инъекции. Если `γ < 0.05` в течение двух итераций, модель отвергает коррекцию, и цикл прерывается.
4. **Жёсткий лимит:** не более 5 итераций.

**7.3.4. Интеграция в поток обработки.** KCA‑цикл запускается в Фазе 2 (Порождение), если начальная уверенность SRG оказалась ниже порога. Он полностью заменяет прежний Inner Critic, обеспечивая более строгую и эффективную коррекцию.

### 7.4. Иерархический HNSW‑индекс

По мере роста архитектуры (множество слоёв, тысячи состояний) линейный FAISS‑индекс, даже оптимизированный, перестаёт справляться. Вводится иерархическая структура.

**7.4.1. Уровни индекса.**
- **Уровень 0 (глобальный):** содержит центроиды всех доменов (из `DomainRegistry` всех слоёв). Используется для быстрой маршрутизации запроса к нужному домену.
- **Уровень 1 (доменный):** для каждого домена — отдельный индекс, содержащий контекстные векторы всех слепков, относящихся к данному домену.
- **Уровень 2 (слой‑специфичный):** внутри домена — подындексы для диапазонов слоёв (1–8, 9–16, 17–24, 25–32), поскольку разные глубины могут активировать разные подпространства состояний.

**7.4.2. Поиск.** При поступлении запроса:
1. На Уровне 0 определяется ближайший домен по косинусному сходству `c_query` с центроидами.
2. На Уровне 1 внутри этого домена ищется ближайшее состояние.
3. При необходимости (если запрос сложный) на Уровне 2 уточняется, какая глубина слоёв наиболее релевантна.

Это обеспечивает логарифмическую сложность поиска даже при миллионах сохранённых состояний.

### 7.5. Полный трёхфазный когнитивный цикл

Все компоненты объединяются в единый цикл обработки запроса.

**Фаза 1: Восприятие и поиск.**
1. Запрос токенизируется и пропускается через таблицу эмбеддингов.
2. Вычисляется контекстный вектор `c_query`.
3. В иерархическом HNSW выполняется поиск: сначала домен, затем ближайшее состояние.
4. При точном совпадении (сходство > 0.95) загружается латентный код `z_stored` и соответствующие матрицы, система переходит к Фазе 3.
5. При частичном совпадении (0.7–0.95) `z` инициализируется интерполяцией, запускается KCA‑цикл (Фаза 2).
6. При отсутствии совпадения запускается холодный старт: `z₀` инициализируется случайно или через гиперсеть, запускается полный KCA‑цикл.

**Фаза 2: Порождение и коррекция (KCA).**
1. Текущий латентный код `z_t` разворачивается в матрицы `ΔW` через слое‑специфичные декодеры.
2. LLM выполняет прямой проход с модифицированными весами и генерирует ответ.
3. Вычисляется `ℒ_KCA(z_t)` по утилитарной функции.
4. Вычисляется градиент `∇_z ℒ` и обновляется `z_{t+1}`.
5. Протокол сходимости проверяет условия остановки. Если не выполнены — возврат к шагу 1.
6. При необходимости (если SRG всё ещё низкая) инициируется создание доменного правила или нового слоя.

**Фаза 3: Исполнение и сохранение.**
1. С финальным латентным кодом `z_T` LLM генерирует окончательный ответ.
2. Вычисляется выходной контекстный вектор `c_out`.
3. Состояние `(c_out, z_T)` проходит валидацию: SRG выше порога, usage_count достаточен, TTL не истёк.
4. При успешной валидации состояние сохраняется в иерархический HNSW, обновляются доменные центроиды.

### 7.6. Ожидаемый результат

По завершении интеграции ЕВА становится полноценной когнитивной архитектурой Fractal Cognitive Fabric, способной:

- Комбинировать знания из разных доменов через State Algebra без экспоненциального роста памяти.
- Гарантированно сходиться к оптимальному ответу за конечное число итераций KCA.
- Эффективно искать релевантные состояния в иерархическом индексе даже при огромном накопленном опыте.
- Проходить полный трёхфазный когнитивный цикл, объединяющий восприятие, порождение и исполнение.

Это завершает построение Единой Вычислительной Архитектуры — самоорганизующейся системы, начинавшей с одного случайного слоя и выросшей в полноценную когнитивную среду.
