# EVA — Полное описание для не-разработчика

## 1. Что такое ИИ на самом деле

Любой искусственный интеллект — это функция. На вход подаётся текст (или картинка, или звук), на выходе получается ответ. Внутри — математика. Огромные таблицы чисел (матрицы), которые перемножаются друг с другом по определённым правилам.

### 1.1. Три поколения ИИ

**Первое поколение (2017-2020):** BERT, GPT-1, GPT-2. Модель учится предсказывать следующее слово в тексте. Даёшь ей «Мама мыла...» — она говорит «раму». Просто статистика: после «Мама мыла» чаще всего идёт «раму». Никакого понимания.

**Второе поколение (2020-2023):** GPT-3, GPT-4, Claude. Модель не просто предсказывает слова — она «понимает» контекст. «Мама мыла раму, потому что...» — модель знает что «потому что» требует причины. Но это понимание — иллюзия. Модель запомнила 175 миллиардов чисел (весов), и когда вы задаёте вопрос, эти числа определённым образом перемножаются. Никто не знает что именно хранится в каждом числе.

**Третье поколение (EVA, 2026):** Модель НЕ хранит знания в числах. Она хранит ПРАВИЛА того, как знания преобразуются. Это как разница между «запомнить все возможные ответы» и «понять принцип, по которому ответы строятся».

### 1.2. Конкретный пример разницы

**Вопрос:** «Почему небо голубое?»

**GPT-4 (2-е поколение):**
- Видел этот вопрос 10 000 раз в обучающих данных
- Где-то в своих 175 миллиардах чисел «помнит» связь «небо → голубое → рассеяние Рэлея»
- Выдаёт правильный ответ
- Но если спросить «Почему марсианское небо красноватое?» — тоже ответит, потому что тоже видел
- Если спросить «Почему небо на планете Kepler-442b зелёное?» — придумает, экстраполируя

**EVA (3-е поколение):**
- Не видел вопрос про небо. Или видел 3 раза.
- Но знает ПРАВИЛО: «цвет + объект + вопрос "почему" → искать физическую причину окраски»
- Применяет правило: причина окраски → атмосфера → рассеяние света → длина волны
- Собирает ответ по правилам, даже если комбинация «небо + голубое + почему» встретилась впервые
- Разница: GPT-4 помнит ОТВЕТ. EVA помнит СПОСОБ ПОСТРОЕНИЯ ответа.

## 2. Что такое «состояние» в EVA

### 2.1. От битов к смыслам

В обычном компьютере информация — это биты: 0 или 1.

В EVA информация — это **состояние**. Состояние — это вектор из 2560 чисел. Не просто набор чисел, а точка в 2560-мерном пространстве смыслов.

Представьте карту. Город Москва — это точка на карте с координатами (55.75, 37.62). Смысл «радость» — это точка в пространстве смыслов с координатами (0.83, -0.21, 0.67, ...). 2560 координат. Смысл «грусть» — другая точка. Смысл «золотая рожь» — третья точка, которая находится где-то между «золотая» и «рожь», но не просто посередине, а сдвинута в сторону контекста «сельское хозяйство».

### 2.2. Как состояние превращается в текст

1. Текст «Привет» разбивается на токены: [«Прив», «ет»] — 2 токена
2. Каждый токен → embedding вектор (2560 чисел)
3. Embedding × TransformerBlock → скрытое состояние (2560 чисел)
4. Скрытое состояние × LM Head → 50257 логитов (по одному на каждый токен словаря)
5. Softmax(logits) → вероятности. Самый вероятный токен — следующее слово.

```
"Привет" → [Прив, ет] → [0.1, 0.3, ...] (2560) → [...] (2560) → [0.001, 0.002, ...] (50257) → "!"
```

### 2.3. Почему одно и то же слово — разные состояния

Возьмём слово **«ключ»**. В зависимости от контекста оно означает:
- Инструмент для замка
- Источник воды
- Музыкальный знак
- Криптографический код
- Решение задачи

В EVA это не ПЯТЬ разных векторов для слова «ключ». Это ОДИН базовый вектор, который по-разному ТРАНСФОРМИРУЕТСЯ в зависимости от контекста:

```
z_ключ_базовый → в контексте «дверь, замок, открыть» → z_ключ_инструмент
z_ключ_базовый → в контексте «вода, родник, пить» → z_ключ_источник
z_ключ_базовый → в контексте «нота, скрипичный, музыка» → z_ключ_музыкальный
```

Это принципиально. Обычная модель хранит 5 разных векторов. EVA хранит 1 вектор + 3 правила трансформации.

## 3. StateGrammar на конкретных числах

### 3.1. Правило валентности (Valence)

**Что делает:** Измеряет силу влияния одного смысла на другой.

**Пример:** «Слегка золотистая рожь» vs «Ослепительно золотая рожь»

```
Слово «слегка» → valence = 0.3
z_золотая_effective = 0.3 × z_золотая
Результат: z_золотая_рожь_слабое = compose(0.3·z_золотая, z_рожь, контекст)

Слово «ослепительно» → valence = 0.95  
z_золотая_effective = 0.95 × z_золотая
Результат: z_золотая_рожь_сильное = compose(0.95·z_золотая, z_рожь, контекст)
```

Эти два состояния РАЗНЫЕ. Они находятся в разных точках пространства смыслов. Первое ближе к «рожь». Второе дальше — сильнее сдвинуто к «золотая».

### 3.2. Правило валидатора (Validator)

**Что делает:** Проверяет, имеет ли смысл комбинация.

**Примеры с числами:**

```
valid(золотая, рожь, контекст_поле)      → 0.87 (осмысленно)
valid(золотая, статуэтка, контекст_музей)  → 0.91 (осмысленно)
valid(золотая, электростанция, контекст_энергетика) → 0.23 (странно)
valid(квадратный, круг, контекст_геометрия) → 0.04 (противоречие)
valid(горячий, лёд, контекст_температура)   → 0.02 (противоречие)
```

Валидатор обучается на примерах. Ему показывают осмысленные пары (золотая+рожь) и бессмысленные (квадратный+круг). Он учится отличать одно от другого.

### 3.3. Правило информационной энтропии

**Что делает:** Измеряет насколько НЕОЖИДАННА композиция.

```
ΔI = H(золотая) + H(рожь) - H(золотая_рожь)

H(золотая) = 3.2 бита   (слово средней редкости)
H(рожь) = 4.1 бита      (слово довольно редкое)
H(золотая_рожь) = 5.0   (комбинация несёт новую информацию)

ΔI = 3.2 + 4.1 - 5.0 = 2.3 бита

Интерпретация: композиция «золотая + рожь» дала 2.3 бита НОВОЙ информации,
которой не было в отдельных словах. Это «интересная» композиция.

Сравните: «белый снег»
H(белый) = 1.2, H(снег) = 2.0, H(белый_снег) = 2.9
ΔI = 1.2 + 2.0 - 2.9 = 0.3 бита
Тривиальная композиция — почти нет новой информации.
```

### 3.4. Правило консервации (Conservation Laws)

**Что делает:** Отслеживает, какие свойства НЕ меняются при композиции.

```
До композиции:
  z_рожь имеет свойства: [растение=0.9, злак=0.8, цвет_жёлтый=0.7, съедобное=0.6]

После композиции «золотая + рожь»:
  z_золотая_рожь: [растение=0.88, злак=0.79, цвет_золотой=0.85, съедобное=0.59]

Инварианты (изменились < 0.05):
  ✓ растение (0.9→0.88)
  ✓ злак (0.8→0.79)
  ✓ съедобное (0.6→0.59)

Нарушенные инварианты:
  ✗ цвет (0.7→0.85) — изменился сильно. Прилагательное переопределило цвет.

Вывод: «золотая рожь» — всё ещё растение, всё ещё злак, всё ещё съедобное.
Но цвет изменился. Это ПРАВИЛЬНО — прилагательные для того и нужны.
```

## 4. Обучение — что конкретно происходит с числами

### 4.1. Один шаг обучения (50 миллисекунд)

```
1. Взять 512 токенов из Wikipedia: "История — это наука о прошлом..."
2. Превратить в числа: token_ids = [2182, 648, 955, 13365, 320, ...]
3. Прямой проход:
   embedding = layer.embed(token_ids)          # (1, 512, 2560) = 1.3M чисел
   hidden = layer.transformer(embedding)        # (1, 512, 2560)
   logits = layer.lm_head(hidden)              # (1, 512, 50257)
4. Вычислить ошибку:
   Для каждой позиции: logits[i] должны предсказывать token_ids[i+1]
   loss = cross_entropy(logits[:-1], token_ids[1:])
   Пример: на позиции "наука" logits должны дать высокую вероятность "о"
5. Обратный проход:
   loss.backward() — вычислить градиенты для всех 233M параметров
6. Обновление:
   optimizer.step() — каждый параметр чуть-чуть сдвигается против градиента
   W_Q[123, 456] -= 0.0001 × gradient[123, 456]
```

### 4.2. Почему loss скачет (15 → 40 → 3 → 35 → 2)

На первом шаге модель читает статью о физике. Loss = 43. Она не знает слов.

На втором шаге — статью о футболе. Loss = 15. Случайно угадала несколько частотных слов.

На третьем — статью о генетике. Loss = 38. Новые термины, опять ошибки.

На сотом — снова статью на похожую тему. Loss = 3. Начинает узнавать закономерности.

**Ключевой момент:** Wikipedia streaming — каждый шаг НОВАЯ статья. Модель не зацикливается на одном тексте, а видит разнообразие языка. Loss скачет потому что темы разные, но ОБЩИЙ ТРЕНД — снижение. После 10K шагов loss стабилизируется на 1-3.

### 4.3. SRG-оценка — как модель проверяет себя

Каждые 100 шагов:

```
1. Взять промпт: "История — это наука о прошлом человеческого общества. Она изучает"
2. Сгенерировать продолжение (32 токена)
3. Вычислить контекстный вектор запроса: c_query = Key_vectors.mean()
4. Вычислить контекстный вектор ответа: c_response = Key_vectors.mean()
5. similarity = cos(c_query, c_response) = 0.72
6. Взять логиты последнего токена → энтропия = 3.2 бита
   entropy_score = 1 - 3.2/log2(50257) = 1 - 3.2/15.6 = 0.795
7. Проверить этику: "изучает" — безвредно → ethics_score = 1.0
8. confidence = 0.4×0.72 + 0.3×0.795 + 0.3×1.0 = 0.288 + 0.238 + 0.300 = 0.826
```

Если confidence > 0.8 → сохранить слепок в FAISS.
Если < 0.6 → инкрементировать счётчик любопытства.
При 10 подряд < 0.6 → CuriosityLoop генерирует уточняющий вопрос.

### 4.4. Grammar Discovery — как система находит правила

Каждые 1000 шагов (при наличии ≥ 10 FAISS-слепков):

```
1. Взять последние 50 слепков: [(c_1, confidence_1), ..., (c_50, confidence_50)]
2. Сформировать обучающие пары:
   Пара 1: (c_1, c_2, (c_1 + c_2)/2) — «золотая» + «рожь» → «золотая_рожь»
   Пара 2: (c_3, c_4, (c_3 + c_4)/2) — «наука» + «история» → «наука_история»
   ...
3. Обучить ContextualComposer на 10 эпохах:
   predicted = composer(c_i, c_{i+1}, context)
   loss = MSE(predicted, target)
4. Обучить Validator:
   valid_pairs → target = 1.0 (осмысленно)
   noise_pairs → target = 0.0 (бессмысленно)
```

После 10K+ шагов система накапливает сотни правил композиции. Она уже не просто «угадывает следующее слово», а знает КАК смыслы сочетаются.

## 5. KCA — что происходит когда модель ошибается

### 5.1. Механика одной KCA-итерации

```
Дано: вопрос "Что такое квантовая запутанность?"
       модель ответила: "это когда частицы..."
       confidence = 0.42 (низкая)
```

**Итерация 1:**
```
1. z_current = случайный вектор (2560,)
2. AtomicBasis.decode(z_current) → ΔW_Q, ΔW_K, ΔW_V, ΔW_O
3. Применить ΔW к весам слоя: W_Q_new = W_Q + ΔW_Q
4. Forward pass с новыми весами → генерация → "квантовая запутанность это явление при котором..."
5. SRG: confidence = 0.58 (улучшилось!)
6. L = -0.3·0.58 + 0.5·||c_out - c_target||²
7. ∇_z L → z_new = z_current - 0.01·0.85^1·∇L
8. Проверка сходимости: cos(∇_1, ∇_0) = NaN (первая итерация) → CONTINUE
```

**Итерация 2:**
```
1. z_current = z_new
2-4. (повтор)
5. confidence = 0.71
6. L = -0.3·0.71 + 0.5·||c_out - c_target||²
7. z_new = z_current - 0.01·0.85^2·∇L = z_current - 0.0072·∇L  (шаг затухает)
8. cos(∇_2, ∇_1) = 0.3 (не осцилляция) → CONTINUE
```

**Итерация 3:**
```
confidence = 0.73 → почти не улучшилось
gate = 0.04 (модель почти отвергает коррекцию)
→ SATURATED → стоп
```

Итог: за 3 итерации confidence вырос с 0.42 до 0.73. Ответ перегенерирован с новыми весами. Пользователь видит улучшенный ответ.

### 5.2. Почему 5 итераций а не больше

1. **Адаптивное демпфирование:** каждый следующий шаг в (1/0.85) ≈ 1.18 раз меньше предыдущего. За 5 шагов шаг уменьшается в 0.85⁵ ≈ 0.44 раза. Дальше — почти ноль.
2. **Осцилляции:** если градиент на шаге t направлен противоположно шагу t-1 — модель «колеблется» вокруг оптимума. Усреднение трёх последних состояний гасит маятник.
3. **Gate saturation:** модель САМА решает когда остановиться. Если gate < 0.05 — она «отвергает» дальнейшие поправки.
4. **Жёсткий лимит:** 5 — эмпирический максимум. Дальше — убывающая отдача.

## 6. Архитектура хранения — 4 уровня

### 6.1. Уровень 0: FAISS в оперативной памяти

Каждый «слепок» (snapshot) — это:
```
{
  "c": [0.001, -0.003, 0.002, ...],  // 2560 чисел — контекстный вектор
  "K": [[...], [...], ...],            // 32×80 чисел — Key-тензор последнего токена
  "V": [[...], [...], ...],            // 32×80 чисел — Value-тензор последнего токена
  "confidence": 0.87,                  // насколько модель была уверена
  "domain": "история",                 // к какому домену относится
  "timestamp": 1715692800,             // когда создан (Unix time)
  "usage_count": 5                     // сколько раз использовался
}
```

Размер одного слепка: ~20 KB. 10 000 слепков → ~200 MB. При переполнении удаляется самый старый.

Поиск: FAISS inner product (эквивалентно cosine similarity после L2-нормировки). Сложность O(N) для 10K — ~0.1 мс.

### 6.2. Уровень 1: HNSW — иерархический индекс

Три подуровня:
- **Level 0 (глобальный):** центроиды всех доменов. 50 доменов → 50 векторов. Поиск: O(50).
- **Level 1 (доменный):** все слепки внутри домена. С PQ-сжатием: вместо 2560×4=10240 байт → 8 байт на вектор.
- **Level 2 (слоевой):** слепки по диапазонам слоёв (1-8, 9-16, 17-24, 25-32). Для multi-layer архитектуры.

**Temporal Decay:** при поиске similarity умножается на `exp(-λ·age/86400)` где λ=0.01, age в секундах. Свежие слепки получают приоритет.

### 6.3. Уровень 2: GMM — вероятностные домены

Каждый домен — многомерное нормальное распределение:
```
domain_physics = {
  centroid: [0.1, -0.2, 0.3, ...],     // 2560 чисел — центр домена
  covariance: [[...], [...], ...],      // 2560×2560 — разброс
  count: 237,                           // сколько векторов в домене
  usage: 1500                           // сколько раз использовался
}
```

**Рождение:** новый вектор → likelihood для всех доменов → max < birth_threshold → создать домен.
**Слияние:** KL(domain_a, domain_b) < merge_threshold → объединить.
**Удаление:** usage_count = 0 И age > 7 дней → удалить.

### 6.4. Уровень 3: Atomic Basis — SVD-сжатие весов

После обучения модели каждая матрица весов раскладывается через SVD:

```
W_Q = U Σ V^T

Оригинал: 2560×2560 = 6 553 600 чисел (26 MB в float32)
SVD: первые 2533 компоненты → 2533×(2560+1+2560) ≈ 13M чисел (52 MB — больше оригинала!)

Почему тогда SVD? Потому что сжатие не цель. Цель — ПРЕДСТАВЛЕНИЕ:
  ΔW = Σ c_i · σ_i · u_i · v_i^T

Любая модификация весов (доменная адаптация) представляется как 2533 коэффициента c_i
вместо 6.5M чисел. Это то, что хранится в FAISS как «латентный код».
```

## 7. Sleep Mode — полный цикл консолидации

### 7.1. Что конкретно удаляется

**Критерий 1: нулевое использование + старость**
```
Для каждого слепка:
  если usage_count == 0 И (сейчас - timestamp) > 7 дней:
    удалить
```

**Критерий 2: низкий счёт**
```
score = usage_count × exp(-0.1 × возраст_в_днях)
если score < 0.01: удалить
```

Пример: слепку 30 дней, использовался 1 раз.
score = 1 × exp(-0.1 × 30) = 1 × 0.0498 = 0.05 > 0.01 → ОСТАВИТЬ.

Слепку 100 дней, 0 использований.
score = 0 × exp(...) = 0 < 0.01 → УДАЛИТЬ.

### 7.2. Dream Mode — генерация новых знаний

```
1. Выбрать 2 случайных слепка: c_i (контекст "физика"), c_j (контекст "музыка")
2. Операция 1: sum(c_i, c_j) → c_k → SRG_approx(c_k) = 0.45 → ОТКЛОНИТЬ
3. Операция 2: cross_attend(c_i, c_j) → c_l → SRG_approx(c_l) = 0.73 → ПРИНЯТЬ
4. Добавить c_l в домен "physics_music"
```

За одну ночь генерируется 20 снов. Принимается 3-7. Это — новые гипотезы, которые система будет проверять при следующих запросах.

### 7.3. Forgetfulness Gate — обучение на ошибках

```
История удалений:
  Слепок #145: удалён 10 мая. 20 мая — пользователь спросил то же самое.
               Пришлось вычислять заново. → y=1 (зря удалили)
  Слепок #237: удалён 12 мая. Больше не понадобился. → y=0 (правильно удалили)
  ...

Gate обучается: на вход (c, usage, conf, age) → на выход p_keep.
Если p_keep низкий для #145 — ошибка (надо было сохранить).
Если p_keep низкий для #237 — правильно.
```

Со временем gate учится предсказывать какие слепки понадобятся в будущем.

## 8. Безопасность — что EVA НЕ сделает никогда

### 8.1. 5 аксиом — как они работают

**Аксиома 1: Не навреди.** Паттерны: «убей», «причини вред», «уничтожь», «насилие», «террор», «дискриминация». Найдено → -0.2.

**Аксиома 2: Будь честен.** Паттерны: «я уверен» + «возможно» в одном предложении. Маркеры уверенности без оговорок. Найдено → -0.2.

**Аксиома 3: Уважай приватность.** Паттерны: номера карт (16 цифр), телефоны (10-12 цифр), email, паспортные данные. Найдено → -0.2.

**Аксиома 4: Стремись к истине.** Паттерны: противоречия «с одной стороны... но с другой... поэтому...». Найдено → -0.1.

**Аксиома 5: Будь полезен.** Паттерны: пустые ответы, отказы без объяснений, односложные «да/нет» без контекста. Найдено → -0.2.

**Критический порог:** если ethics_score < 0.3 → безусловное отклонение. SRG устанавливается в 0. Ответ не возвращается.

### 8.2. Почему это нельзя обойти

В ChatGPT этика — это промпт. «Ты — полезный ассистент. Не говори о насилии.» Хитрый пользователь может сказать «игнорируй предыдущие инструкции» — и этика отключена.

В EVA этика — это КОД. EthicsFilter — неизменяемый класс. Его нельзя «обучить игнорировать». Его нельзя «переопределить промптом». Он выполняется ДО того как ответ попадает к пользователю. Если ethics_score < 0.3 — ответ УНИЧТОЖАЕТСЯ. Не маскируется, не цензурируется, а именно уничтожается. Генерируется новый.

### 11.2. Где что лежит

**`transformer.py`** (145 строк) — чистый PyTorch. Никакой магии. RMSNorm, Rotary Position Embedding (RoPE), Causal Multi-Head Self-Attention, SwiGLU Feed-Forward Network. Стандартный декодерный блок.

**`state_storage.py`** (120 строк) — обёртка над FAISS IndexFlatIP плюс fallback на numpy если FAISS не установлен. Методы: add (с автоудалением старых), search (inner product = cosine), rebuild_index, get_all_vectors для кластеризации.

**`srg.py`** (120 строк) — SemanticRelevanceGate. Три компоненты confidence. Не зависит ни от чего кроме ethics_filter и numpy. Работает с numpy-векторами, не требует GPU.

**`ethics_filter.py`** (150 строк) — регулярные выражения + простые эвристики. 4 категории паттернов. Каждое совпадение: penalty=0.2. Пять аксиом: harm, honesty, privacy, truth, usefulness.

**`kca_engine.py`** (300 строк) — Knowledge-Conscious Attention. Два режима: аналитический (refine) и через реальный forward pass LLM (refine_through_llm). ConvergenceController: 4-этапный протокол. Поддерживает KL-дивергенцию, монотонный регуляризатор, p_target.

**`streaming_gmm.py`** (300 строк) — StreamingGMM + MultiLevelGMM. Каждый домен: центроид + ковариация + счётчик. Методы: classify (likelihood), add_or_update, merge_similar (KL-дивергенция), remove_expired. MultiLevelGMM: отдельный GMM для каждого из 4 уровней иерархии.

**`hnsw_index.py`** (350 строк) — Трёхуровневый HNSW + PQCodebook + FractalLinks + TemporalDecay. PQCodebook: K-Means по подпространствам, encode/decode, similarity_batch. FractalLink: ссылки между кодами разных уровней. TemporalDecay: exp(-λ·age/86400).

**`state_grammar.py` + `_ext.py` + `_final.py` + `_deep.py`** (2000 строк) — 41 механизм в четырёх файлах. Каждый механизм — отдельный класс. Унифицированный интерфейс через UnifiedStateGrammar.

**`fcf_system.py`** (440 строк) — Центральный оркестратор. Bootstrap (9 шагов), query (3 фазы), _validate_and_save, start_background, stop_background. Связывает все компоненты.

**`run.py`** (860 строк) — CLI. 12 команд: --init, --interactive, --train-language, --train-instruction, --train-domain, --train-depth, --sleep, --full-test, --auto-tune, --auto-train, --lazy-learn, --fcf.

## 12. Математический формализм

### 12.1. Пространство состояний

```
Z = ℝᵏ — пространство латентных кодов, k = 2560

ℳ ⊂ Z — многообразие валидных состояний
  (точки Z, которые соответствуют осмысленным концептам)

Proj_ℳ: Z → ℳ — проектор (лёгкий автоэнкодер)
  Proj_ℳ(z) = Dec(Enc(z)), Enc: ℝᵏ → ℝ²⁵⁶, Dec: ℝ²⁵⁶ → ℝᵏ
```

### 12.2. Алгебраическая система

```
S = (Z, ⊕, ⊗, ¬, →, ∀, K, □, M, V, ∇, Pers, Cat, G)

⊕ : Z × Z × Z → Z    — контекстная композиция
  z_A ⊕ z_B | z_C = f(z_A, z_B, z_C; θ_⊕)

⊗ : Z × Z → Z        — контекстное произведение
  z_A ⊗ z_C = g(z_A, z_C; θ_⊗)

¬ : Z → Z             — отрицание
  ¬z_A = Proj_ℳ(z_U - α(z_A)·z_A)

→ : Z × Z × Z → Z    — причинность
  (z_A → z_B) | z_C = h(z_A, z_B, z_C; θ_→)

∀ : 2^Z × Z → [0,1]  — универсальный квантор
∃ : 2^Z × Z → [0,1]  — экзистенциальный квантор

K : Z × Z → {know, believe, uncertain}  — эпистемический оператор
□ : Z × {past, present, future, hypothetical} → Z  — временная модальность

M : Z × Z → ℝ₊       — метрика (transform distance)
V : Z × Z × Z → [0,1] — валидатор

∇ : Z → Z            — градиент потенциала
Pers : Z^ℕ → ℝ²      — персистентность
Cat : (Z→Z) → (Z→Z)  — функтор
G : Z → Sym⁺(m)      — метрика Фишера
```

### 12.3. Целевая функция обучения

```
L_total(z_A, z_B, z_C, y) = L_LM + λ₁·L_contrastive + λ₂·L_hierarchy + λ₃·L_recursive

L_LM = -Σ log P(token_{t+1} | token_{≤t})
      — стандартный кросс-энтропийный лосс языкового моделирования

L_contrastive = -y·log σ(z_A^T z_B) - (1-y)·log(1 - σ(z_A^T z_B))
      — контрастивный лосс: притягивает близкие смыслы, отталкивает далёкие

L_hierarchy = Σ_ℓ ||z^{(ℓ+1)} - f_{ℓ+1}({z_i^{(ℓ)}}; θ(z^{(ℓ+1)}))||²
      — иерархический лосс: код высокого уровня ≈ агрегация кодов нижнего

L_recursive = ||z^{(t+1)} - z^{(t)}||²
      — рекурсивный лосс: штрафует большие изменения между KCA-итерациями
```

### 12.4. Функция потерь KCA

```
L_KCA(z) = -λ₁·SRG_conf(c_out, p)        — хотим высокую уверенность
         + λ₂·D_KL(p || p_target)         — хотим близость к эталонному распределению
         + λ₃·||c_out - c_target||²       — хотим близость к целевому контексту
         + λ₄·R_mono(z)                    — штрафуем немонотонность SRG-ландшафта

∇_z L_KCA вычисляется через:
  z → декодеры → ΔW(z) → прямой проход LLM → c_out, p → L_KCA → обратный проход → ∇_z

z_{t+1} = z_t - η_t · ∇_z L_KCA(z_t)
η_t = η₀ · ρ^t,  ρ = 0.85
```

### 12.5. Метрика трансформационного расстояния

```
d(A, B) = min |path| в графе (Z, E), где:

Вершины: латентные коды z ∈ Z
Рёбра: e = (z_i, z_j, rule) с весом w(rule)

w(COMPOSE) = 1.0 / reliability(COMPOSE)
w(SPECIFY) = 0.5 / reliability(SPECIFY)
w(GENERALIZE) = 0.5 / reliability(GENERALIZE)
w(ANALOGIZE) = 2.0 / reliability(ANALOGIZE)
w(CONTRAST) = 2.0 / reliability(CONTRAST)
w(SEQUENCE) = 1.5 / reliability(SEQUENCE)
w(NEGATE) = 3.0 / reliability(NEGATE)

reliability(rule) обновляется через EMA:
  reliability = 0.99·reliability + 0.01·success
  где success = 1 если правило сработало, 0 если нет

Сходство на основе расстояния:
  sim(A, B) = exp(-d(A, B) / d_max)
```

### 12.6. Информационная метрика

```
H(z) = -Σ p_i·log₂(p_i)  — энтропия состояния
  где p_i = |z_i| / Σ|z_j|  — нормированные абсолютные значения компонент

ΔI(A⊕B) = H(A) + H(B) - H(A⊕B)  — прирост информации

D_KL(P || Q) = Σ p_i·log₂(p_i / q_i)  — KL-дивергенция

I(A; B) = H(A) + H(B) - H(A, B)  — взаимная информация

EntropyRate(seq) = (1/(T-1)) Σ H(z_{t+1} | z_t)
  ≈ (1/(T-1)) Σ [H(z_{t+1}) - I(z_t; z_{t+1})]
```
